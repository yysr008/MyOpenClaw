# HEARTBEAT.md - 定期任务

## 1️⃣ 财经汇报（每 6 小时一次）
- **触发条件**：心跳检测时自动执行或手动说 “汇报财经”
- **执行步骤**：
  1. 运行 `node financial_report.js` 抓取全球财经数据  
  2. 生成投资建议报告（包括理由、具体建议、风险等级）  
  3. 通过 Telegram 向 Dominus‑主人汇报
- **状态记录**（手动更新）：
  - ✅ 最近一次执行：`{{LAST_FINANCIAL_RUN}}`  
  - ✅ 上次汇报内容：`{{LAST_FINANCIAL_SUMMARY}}`  

## 2️⃣ 深圳天气汇报（每小时整点 & 半点）
- **触发条件**：心跳检测或用户手动说 “深圳天气”
- **执行步骤**：
  1. 使用 `weather` 工具查询深圳实时天气  
  2. 将以下格式的简 brief 消息发送至 Telegram  
     > 当前深圳天气：☀️ {{TEMP}}°C，{{WIND}}，预计 {{NEXT_2H}} 降至 {{LOW_TEMP}}°C
  3. 更新状态记录
- **状态记录（手动更新）**：
  - ✅ 最近一次查询：`{{LAST_WEATHER_TIME}}`  
  - ✅ 天气简报：`{{LAST_WEATHER_SUMMARY}}`

## 4️⃣ 备注
- **手动触发**：若系统未自动执行，可在聊天中直接说 **“汇报财经”** 或 **“深圳天气”** 来强制触发。
- **状态更新**：每次完成任务后，请在对应的 *状态记录* 区域填入实际时间戳或简报内容，以便后续审计。
- **版本控制**：建议每次修改后提交一次 git（`git add HEARTBEAT.md && git commit -m "Update heartbeat schedule"`），便于追踪变更历史。

### 使用示例

1. **财经汇报**  
   - 手动触发：`@bot 汇报财经`  
   - 自动触发：心跳检测到 `generateReport` 标记时执行。

2. **天气查询**  
   - 手动触发：`@bot 深圳天气`  
   - 自动触发：心跳检测到 `weather` 标记时执行。

3. **查看检查清单**  
   - 只需在 Telegram 中发送 `查看检查清单`（或直接查看 `HEARTBEAT.md`），即可看到当前进度。
